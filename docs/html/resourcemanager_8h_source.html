17 /*
18 Design of ResourceManager:
19 The purpose is to add cache magagement to external buffer owned by hwcLayer
20 to avoid import buffer and glimage/texture generation overhead
21 
22 1: the ResourceManager is owned per display, as each display has a
23 separate
24 GL context
25 2: ResourceManager stores a refernce of external buffers in a vector
26  cached_buffers, each vector member is hash map.
27  The vector stores history buffer in this way, vector[0] is for the current
28  frame buffers, vector[1] is for last frame (-1) buffers, vector[2] is -2
29  frames buffers etc. A constant (currently 4) frames of buffers is stored.
30  When a buffer refernce is stored in vector[3] and it is not used in the
31  current frame present, it will go out of scope and be released.
32  If a buffer is fetched from map, it will always re-registered in vector[0]
33 map.
34 3. By this way, drm_buffer now owns eglImage and gltexture and they
35  can be resued.
36 */
